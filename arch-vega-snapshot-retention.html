1 · The Concept, In My Own Words

The Concept, In My Own Words

The Vega Lead page is an archive of historical vega-flow leading indicators, captured once per trading day at market close. Right now, you can access past snapshots via a date dropdown, but it's not obvious—you only see one day at a time, and the UI defaults to the most recent. You want a clearer way to browse and retain 30 days of history, so you can spot trends and review how vega positioning evolved over a month without manually hunting through a small dropdown.

Why it matters: Trading decisions benefit from seeing patterns across days. A 30-day rolling window gives you enough data to spot vega-flow regime shifts, without the cognitive load of maintaining years of history.

2 · What the Code Does Today

What the Code Does Today

Storage:

  • Intraday vega-flow (live): Stored in Redis list chain:vegaflow:yyyyMMdd, sampled ~60s during market hours (09:10–15:35 IST). TTL 30 hours.
  • Daily snapshot (archived): At market close (15:30 IST), ChainAnalysisService.SnapshotDailyVegaFlowIfDueAsync() copies the day's intraday points from Redis into Postgres table VegaFlowDailySnapshots (one row per trading day). No auto-expiry; manual deletion only.

API:

  • GET /api/v1/analysis/vega-flow → today's intraday Redis series (live data, updated ~60s)
  • GET /api/v1/analysis/vega-flow/history-dates → list of all archived dates in Postgres, most recent first
  • GET /api/v1/analysis/vega-flow/history/{date} → one day's complete snapshot (VegaFlowPoint[])

UI (VegaLeadView):

  • Date dropdown (lines 62–74) lists all available snapshots, defaults to most recent on first load.
  • Clicking a date fetches and renders its snapshot as a two-axis overlay chart (CE/PE vega left, NIFTY spot right).
  • Snapshots are independent; selecting one doesn't auto-update when new dates arrive.

Current flow diagram:

Market Hours (09:10-15:35 IST, Mon-Fri)
   ↓ every ~60s
  MarketDataWorker samples vega-flow
   ↓
  ChainAnalysisService.AppendVegaFlowPoint()
   ↓
  Redis chain:vegaflow:yyyyMMdd (list, 30h TTL)
   ↓
   ↓ At 15:30 IST (market close)
   ↓
  SnapshotDailyVegaFlowIfDueAsync()
   ↓
  Postgres VegaFlowDailySnapshots (permanent, no expiry)

  Date dropdown + interactive chart

10 · Implementation Sketch (Non-Binding)

Implementation Sketch (Non-Binding)

Step 1: UI Improvement (Low Risk)

  1. Enlarge and reposition the date dropdown in VegaLeadView (e.g., center it below title, make font larger).
  2. Rename it from generic 'Select date' to 'Browse trading history (30 days)'.
  3. Add a help text: 'Snapshots are archived once per trading day at 15:30 IST. Today's intraday data is in the Option Chain view.'
  4. Test that the dropdown still works with long date lists and renders responsively.

Step 2: Backend Filtering (Medium Risk, if 30-day limit desired)

  1. Modify AnalysisController.GetVegaFlowHistoryDates() to filter the returned list: WHERE TradingDate >= DateOnly.FromDateTime(DateTime.Now.AddDays(-30)).
  2. Update API docs to clarify the 30-day window.
  3. Test that the endpoint returns ~20–22 dates for a month of trading data.
  4. Test edge cases: first week of data (fewer than 30 days available), after holidays (non-consecutive dates).

Step 3: Optional Cleanup Job (Higher Risk, if hard-delete desired)

  1. Create a new hosted service VegaSnapshotCleanupService in ThetaDesk.Workers (or ThetaDesk.Api if workers not available).
  2. Implement a periodic task (e.g., every 24h at 23:59 IST) that deletes rows: DELETE FROM VegaFlowDailySnapshots WHERE TradingDate < Now - 30 days.
  3. Make it idempotent (safe to run multiple times per day).
  4. Add logging and optional alert if deletion fails.
  5. Validate timezone correctness: TradingDate is a DateOnly in IST, so ensure the comparison is in IST too.

Step 4: Optional UI Richness (Lower Priority)

  1. If operators request deeper history browsing, add a new endpoint: GET /api/v1/analysis/vega-flow/history-range/{fromDate}/{toDate} to fetch multiple snapshots at once.
  2. Add a 'History Grid' or 'Timeline' view in VegaLeadView that shows mini-charts or a trend line across all 30 days.
  3. Use pagination or lazy-loading to avoid fetching all 30 snapshots on page load.

Step 5: Testing & Validation

  1. Unit test the date filtering logic and cleanup query with various date ranges and timezone scenarios.
  2. Integration test: mock market close, verify snapshot is written, verify it appears in history-dates within 2 minutes.
  3. Manual test: open VegaLeadView at 14:00 IST (before close), verify today is not in dropdown; come back at 16:00 IST (after close), verify it appears.
  4. Cleanup validation: manually insert old rows, run cleanup job, verify deletion and no newer rows are harmed.
